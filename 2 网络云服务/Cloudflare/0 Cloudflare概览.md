# Cloudflare优点

* Cloudflare R2 最大的优势是 **不收取数据下载（egress）费用**，这在传统对象存储（如 AWS S3）中往往是最贵的一项费用。
* R2 能很好地配合 Cloudflare 的全球 **CDN** 网络，访问速度快、延迟低。特别是配合设置合理的缓存策略，访问体验可媲美付费 CDN。
* 支持 S3 协议的标准 API，你可以用常见 SDK（例如 aws-sdk、boto3 等）来上传/读取，这对于自动化上传或图床工具对接很方便。
* Cloudflare R2 提供每月一定的 **免费存储和请求额度**（如 10GB 存储 + 读写操作免费额度），对于个人博客、小项目图床足够。

## 注意：自定义域名要求与你 DNS 配置有关！

如果要用自己域名（如 `img.yoursite.com`）访问 R2，通常需将域名托管到 Cloudflare 并正确设置 DNS，这对某些人来说可能需要多一步操作。

# 免费版使用额度

| 指标                 | 免费额度（每月）                           |
| ------------------ | ---------------------------------- |
| 存储空间               | 10 GB                              |
| Class A 操作（写入/列出等） | 1,000,000 次，超过后是 **每百万次 ~ $4.50**。 |
| Class B 操作（读取对象）   | 10,000,000 次，超过后是 **每百万次 ~ $0.36** |
| 出站流量（egress）       | 无限免费                               |
|                    |                                    |

> 注意：免费额度每月循环重置。
> 注意：合理设置 CDN 缓存策略能极大降低读取计数！

## ✅ Class A 操作（写/修改相关）

这类操作主要是会改变数据状态的，比如：

- 上传文件（PutObject）    
- 创建 Bucket    
- 列出对象（ListObjects）    
- Multipart Upload / Complete Upload    
- 修改 CORS、Lifecycle 设置 等    

## 📖 Class B 操作（读取相关）

这类操作主要是跟读取文件、获取 metadata 有关，比如：

- 下载文件（GetObject）    
- 获取对象头信息（HeadObject）    
- 查询 Bucket 信息 等
